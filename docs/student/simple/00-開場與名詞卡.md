# 開場 5 分鐘 ＋ 一張名詞卡（看不懂就翻回這頁）

> 給今天的你：**不用寫程式、不用懂 AI**。照著速查卡點，看到跟圖一樣就對了。
> 這頁不用背。等等動手時遇到哪個詞，翻回來看一句就好。

---

## 一句話講完今天

> **AI（像 ChatGPT）很會講話，但有兩個毛病：①不知道「你公司」的事、②不確定時會很有自信地亂編。
> 今天就是把這兩個毛病補起來——做一個「只講你公司的事、而且不亂編」的客服機器人。**

---

## 一個比喻就懂：閉書考 vs 開書考

- **AI 自己答＝閉書考**：會的就答，不會的就*硬掰*（這叫「幻覺」）。
- **給它你公司的文件再答＝開書考**：先翻你的 FAQ／SOP／合約範本，再照著回答。**這就是今天的核心。**

> 你的專業版比喻：
> - 會計：AI 亂編 ＝「**沒有憑證就認列**」；開書考 ＝「先調出傳票再下結論」。
> - 法律：AI 亂編 ＝「**沒查判例就給意見**」；開書考 ＝「先翻條文/契約再回答」。
> - 資訊：就是 **RAG**（回答前先檢索你的文件）。

## 今天你會做 vs 只用看的

| | 內容 | 誰做 |
|---|---|---|
| **你親手做** | ① 用 ChatGPT 做一個「只會聊天」的客服 → 親眼看它亂編 | 👉 你 |
| **你親手做** | ② 用 Dify 給它公司文件 → 變成「答得準、還附出處」的客服 | 👉 你 |
| 看老師示範 | 讓它會**查訂單**、接到 **LINE**、處理**複雜關係問題（圖譜）** | 👀 看就好 |

> 心法：**先看它出包（①），再看補起來有多有感（②）**。後面進階的，回公司交給工程師或改天再玩。

## 名詞卡（只有 6 個，每個一句話）

| 你會看到 | 一句話白話 |
|------|------|
| **AI／大型語言模型** | ChatGPT 背後那個「很會接話」的東西。很會講，但不懂你公司。 |
| **幻覺** | 它不知道答案時，會**很有自信地編一個**。今天 Stage 1 你會親眼看到。 |

| **Chatbot（聊天機器人）** | Dify 裡最單純的一種——上傳文件就會照文件回答。**今天你建的就是這種。** |
| **工具 / LINE**（今天只看） | 讓機器人不只會講、還會**做事**（查訂單）或**接到 LINE** 上。老師示範，你先不用碰。 |
