講師口白稿（超白話版）— 照著唸就不會太深

對象：會計 / 法律 / 資訊顧問等級，聰明但非 AI 背景。原則：先比喻、再（選擇性）給術語；能不講的術語就不講。 標 🖐️＝學員動手；標 👀＝只示範、學員不做。全程不要求學員打任何指令。

0. 開場（3 分）

「今天不考試、不寫程式。你只要會複製貼上、點選，就能做出一個公司客服機器人。 先講一句話你就懂今天在幹嘛——ChatGPT 這種 AI 很會講話，但它①不知道你公司的事、②不確定時會很有自信地亂編。今天就是把這兩個毛病補起來。」

開書考比喻（務必講）：

「AI 自己回答＝閉書考，會的就答、不會就硬掰。我們今天做的是開書考——先把你公司的文件塞給它，它照著翻、照著答。」 - 對會計：「就像沒憑證不能認列——沒有文件就不要它亂講。」 - 對法律：「就像沒查到條文/判例就不給意見。」 - 對資訊顧問：「這在業界叫 RAG，先檢索再回答，今天你會親手做一次。」

1. 自評 worksheet（5 分，🖐️）

「先憑直覺勾一勾：你公司資料多不多、要不要接 LINE/網站、有沒有法遵限制、有沒有工程師。等等看完你會知道自己該停在哪一段最划算。」（用 student-worksheet-self-assessment.md）

2. Stage 1：親眼看它亂編（15 分，🖐️ 速查卡①）

「打開 ChatGPT，把我給的那段貼進去，它就假裝是一家餐廳客服。然後問它『20 人包場多少錢、能不能幫我線上訂位付訂金』。」 （等大家問完） 「看到沒？它很自信地編出一堆價格——但這些數字我從來沒給過。而且它根本不能真的幫你訂位。 如果這是你公司的客服，客人被報錯價、又辦不了事。這就是天花板：很會講，但不懂你公司、不能做事。」 - 一句收：「記住這個『亂編』的感覺，等等你會看到怎麼一次補掉。」

3. Stage 2：給它公司文件就不亂編（35 分，🖐️ 速查卡②）

「現在換 Dify——一個不用寫程式的工具。我們分兩步：先做一個『文件櫃』，再做一個『照文件回答的機器人』。」

Part A 文件櫃（知識庫）：

「Knowledge → Create → 把兩份文件拖進去 → 下一頁什麼都不用改、直接存 → 等它變 Available。這一頁上面那些設定是工程師在調的，今天用預設就很好，別碰。」

Part B 機器人（Chatbot）：

「Studio → Create → 從空白 → 選 Chatbot 這個最單純的（不要選旁邊的 Agent，那個是進階的）。貼一句指令、把剛剛的文件櫃掛上去。」 「右邊問它『你們的退貨原則是什麼』——看，它照文件回答，下面還列出是哪份文件（出處）。這就是開書考。跟剛剛 ChatGPT 亂編完全不一樣。」 - 對法律/會計：「重點是有出處、可追溯——它不再自己腦補。」

4. 👀 示範段：讓它更強（20 分，學員只看）

「接下來這幾樣你先不用動手，我做給你看，知道『長怎樣、什麼時候值得做』就好，回公司交給工程師或改天自己玩。」 1. 會查訂單（工具）：切到我先建好的進階機器人，問「訂單 A1001 到哪了」→ 它真的去查、回出貨狀態。「這叫給它一個工具，讓它會做事，不只會講。」 2. 接到 LINE：手機傳訊給我們的 LINE 官方帳號 → 機器人在 LINE 上回答。「這樣客人不用進後台，直接用 LINE 就能問。」 3. 複雜關係問題（圖譜）：用「檢索測試」頁對照——問「跟這支豆同供應商的還有哪些」，一般作法漏、加了『關係圖』就答得齊。「有大量『跟 X 有關的還有哪些』這種問題，才值得上這個；一般 FAQ 不用。」 （若現場網路/服務不穩，直接投影備援截圖 docs/instructor/img/stage3/，效果一樣。）

5. 收尾：你該停在哪、怎麼開始（15 分）

「回到 worksheet：多數公司停在 Stage 2（Dify）就非常夠用——CP 值最高。要接很多通路、或有大量關係型問題，再往上。」 - 三做法比喻（要就講）：「ChatGPT＝住飯店（拎包入住、不能改）；Dify＝租屋自己佈置（彈性大、最划算）；全自建＝自己蓋房（最貴最自由）。」 「行動：這週先把公司 FAQ/SOP 整理成幾份文件；一個月內用 Dify 做個內部試用；三個月再決定要不要接通路、上進階。」

現場常見狀況（助教備忘）

  • 學員找不到 Agent 的 Tools？→ 超白話版本來就不做工具，請他們用 Chatbot 就好。
  • 機器人答「不確定」？→ 問題可能不在文件裡；換問「退貨原則/運費/付款方式」這種文件有的。
  • 有人問向量/webhook/Docker…→ 「那是工程師的細節，今天用不到，我課後給你進階講義。」（指到 docs/student/ 詳版與 student-line-setup.md）
