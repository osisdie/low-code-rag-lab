# 自評 worksheet：我公司該選哪種 AI 客服做法？

> 開場用。花 5 分鐘填，再跟旁邊的人對一下。

---

## 第一部分：四維評估（勾選）

**1. 你公司的客服文件大約幾頁？**
- [ ] <50 頁
- [ ] 50–1,000 頁
- [ ] 1,000–10,000 頁
- [ ] 10,000+ 頁

**2. 你需要把 bot 串到哪些通路？（可複選）**
- [ ] 沒有，只在內部用
- [ ] 網站 widget
- [ ] LINE 官方帳號
- [ ] FB Messenger
- [ ] Email
- [ ] CRM / 工單系統
- [ ] 語音

**3. 你的資料有合規限制嗎？**
- [ ] 沒有（產品 FAQ 等公開資料）
- [ ] 有（含客戶資料 / 金流 / 法遵）
- [ ] 高度敏感（金融 / 醫療 / 政府）

**4. 你的團隊有專職工程師嗎？**
- [ ] 沒有
- [ ] 1 位（兼任）
- [ ] 2+ 位（專職）
- [ ] 完整 ML team

---

## 第二部分：算出你的落點（講師帶大家算）

```
全部勾左邊 1-2 項        → ChatGPT 做法（Stage 1）
2-3 題勾中間             → Dify 做法（Stage 2）
3+ 題勾右側、有合規/工程師 → 工程做法（含進階圖譜，Stage 3 概念）
```

---

## 第三部分：對照四個真實案例

| 案例 | 公司 | 資料量 | 落點 | 為什麼 |
|------|------|--------|------|--------|
| A | 餐飲連鎖 30 店 | ~20 頁 | ChatGPT | 資料少、只在 IG DM 回、店長一下午建得起來 |
| B | 跨境電商 SaaS | ~800 頁 | Dify | 多語系 + 多通路、客服 burnout、需 routing |
| C | 銀行私人銀行 | ~50,000 頁 | 工程 | 合規不能上 OpenAI、需身分核對、答錯成本極高 |
| D | 線上教育平台 | ~1,500 頁 | Dify → 工程 | 起步 Dify 驗證、半年後破 10 萬用戶再升級 |
