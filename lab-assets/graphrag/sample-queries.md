# 對照查詢：向量 RAG vs 圖譜 RAG

> Stage 3 demo 的「武器」。同一個問題，分別問 **Dify 內建知識庫（純向量）** 與
> **外掛 GraphRAG（向量 + 圖譜）**，現場對照誰答得全。
>
> 資料來源：`../knowledge-base/coffee-catalog-relationships.md`

---

## 為什麼向量會輸？

純向量檢索是「**找語意最相似的片段**」。每個商品在文件裡各自一段，
當問題需要「跨多個商品、沿著共用的供應商／產線／政策把它們連起來」（多跳、關聯），
向量通常只撈到問題裡**直接提到**的那一段，撈不到「**和它有關係的其他商品**」。
圖譜檢索是「**沿著實體之間的邊走**」，所以能一次撈齊。

## 題組（建議現場依序 demo）

### Q1 — 暖身（向量也答得出）：單一商品屬性
**問**：耶加雪菲是什麼烘焙度、什麼風味？
- 向量：✅ 答得出（答案就在同一段）
- 圖譜：✅ 也答得出
- **教學點**：先讓學員看到「簡單題兩邊都行」，建立基準。

### Q2 — 同供應商（1 跳）
**問**：和「耶加雪菲」**同一個供應商**的，還有哪些咖啡豆？
- 純向量：⚠️ 常只回耶加雪菲本身，或漏掉西達摩
- 圖譜：✅ 沿 商品→供應商(Sidama Union)→其他商品，回出 **P2 西達摩日曬**
- **預期答案**：西達摩 日曬（P2）

### Q3 — 同產線且同政策（2 條件 join，多跳）★ 主秀
**問**：哪些商品和「冷萃黑咖啡」**在同一條產線、而且退換貨政策也相同**？
- 純向量：❌ 多半只回冷萃黑咖啡那段，答不出「還有誰跟它同條件」
- 圖譜：✅ 沿 商品→產線(L4) 與 商品→政策(C) 兩條邊取交集
- **預期答案**：氮氣冷萃 拿鐵（P8）（同為 L4 冷萃線 + 政策 C）

### Q4 — 斷貨影響（供應商→商品→促銷組合，連鎖多跳）★ 商業感最強
**問**：如果供應商「La Esperanza」突然斷貨，會**影響哪些商品和哪些促銷組合**？
- 純向量：❌ 難以聚合，常漏掉「組合」這層
- 圖譜：✅ 供應商→(P3 薇拉甜橙, P4 低因)→組合(B2 入門淺焙組含 P3)
- **預期答案**：商品 P3 薇拉甜橙、P4 哥倫比亞低因；促銷組合 **B2 入門淺焙組**

### Q5 — 組合溯源（組合→商品→豆源產區）
**問**：「夏日冷萃組」用到哪些**產區**的豆子？
- 純向量：⚠️ 可能只回組合內容，答不到產區那一跳
- 圖譜：✅ 組合(B4)→(P7,P8)→豆源供應商→產區
- **預期答案**：P7 來自瓜地馬拉（Finca El Injerto）、P8 來自台灣阿里山合作社

## 用 curl 直接測 adapter（不經 Dify）

```bash
curl -s -X POST http://localhost:8000/retrieval \
  -H "Authorization: Bearer lab-graphrag-secret" \
  -H "Content-Type: application/json" \
  -d '{
        "knowledge_id": "coffee-graph",
        "query": "和冷萃黑咖啡同一條產線、且退換貨政策也相同的商品有哪些？",
        "retrieval_setting": { "top_k": 8, "score_threshold": 0.2 }
      }' | python -m json.tool
```
回應應包含一筆 `records`，`content` 內含沿著「產線 L4」「政策 C」抓出的 P7/P8 關係脈絡。

## demo 收斂金句

> 「向量 RAG 是『找像的句子』，圖譜 RAG 是『走通的關係』。
> 客服知識裡大量是『同供應商／同政策／同方案』這種關係問題 —
> 這就是為什麼進階導入時，圖譜不是炫技，是補向量的盲區。」
