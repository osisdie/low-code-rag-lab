Stage 0 · 觀念地基

動手前，先記一句話

AI 客服 = 一個很會講話、但①不知道你公司的事、②不確定時會亂編的模型。
今天三段，就是一層一層把這兩個毛病補起來。
💡 一個比喻：AI 自己答 =「閉書考」，會的就答、不會的硬掰；RAG =「開書考」，先翻你給的文件再答。今天就是把閉書考變開書考，再讓它會查訂單。
補「私有資料」

RAG（開書考）

回答前先翻你的文件再答 → 不亂編、能用公司資料。Stage 1 / 2。

補「即時動作」

Tool 工具

不只會聊天，還能查訂單、轉真人 → 補「不會做事」。Stage 2。

補「關係問題」

知識圖譜

沿「實體＋關係」走，答得出「跟這個有關係的還有哪些」。Stage 3。

Stage 1 · No-Code

用 ChatGPT 做 AI 客服

WHY 為什麼

  • 門檻最低、最快有成就感
  • 建立「上傳資料→AI 會答」直覺
  • 親手撞天花板，自然帶出下一段

HOW 怎麼做

  • 建 Project → 上傳餐廳 FAQ
  • 貼新手 prompt（4 句不要亂答）
  • 兩人互測：能答 / 答不出 / 挑釁
Stage 1 · 天花板

它出不了 ChatGPT 視窗

  • 沒辦法嵌到網站 / LINE
  • 不能查資料庫（無 tool calling）
  • 文件大就慢、太大會被截斷
  • 資料會進 OpenAI
👉 下一段，用 Dify 一次解掉。
14:50–16:00 · 動手
Stage 2 · Low-Code

用 Dify 串 LINE（向量 RAG）

WHY 為什麼

  • 一次補齊：通路 + 知識庫 + 查訂單
  • 不必養工程團隊就能上線
  • 用的是 向量 RAG＝比「意思像不像」找答案（不只比關鍵字）

HOW 怎麼做

  • 把公司文件切小段存進去（切多細、撈幾段可微調）
  • 貼進階 prompt → 它會查訂單、會轉真人
  • 接到網站小工具 / LINE
16:15–17:15 · 講師示範
Stage 3 · Advanced

向量的盲區：關係 / 多跳問題

💡 比喻：向量像「Google 找相似句子」；圖譜像「查族譜 / 關係網」——沿著線把所有相關的人事物一次抓齊。

🟠 向量 RAG

「找像的句子」。問「跟這支豆同供應商的還有哪些？」常只回你問到的那一支。

🟢 圖譜 RAG

「走通的關係」。沿 商品─供應商─產線─政策 的邊走，一次撈齊相關實體。

圖譜不是炫技，
是補向量的盲區。
Stage 3 · 架構

同一個機器人，問兩個資料來源

回答時同時查：①向量（找像的句子）②圖譜（走關係）。Dify 留了一個標準插座，任何圖譜服務都能插上來。（工程細節，看懂概念即可）
